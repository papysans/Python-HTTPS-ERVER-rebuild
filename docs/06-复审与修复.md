# 第二轮复审与修复（复审重构产物本身）

第一轮的四方审查（3 个 Claude + 1 个 Codex）审的是**原始**那份刻意写坏的代码，产出了 30 条问题、指导了重构。但重构后的代码是**新写的、还没有任何人审过**。所以我们又发起了第二轮，专门审重构产物本身。

> **这一轮最重要的意义**：它证明了「审查原始代码 + 通过测试」不等于「重构是对的」。重构引入的 bug，只有再审一遍才抓得到——包括一个真实的路径穿越漏洞。

## 一、这一轮怎么做的

- **审查者**：Codex（GPT-5 家族）。请求的是 xhigh 推理强度，Codex 侧实际报告用的是 **high**（如实记录，未美化）。
- **对象**：只审重构产物（`config/http_core/sessions/handlers/http_server` + 测试），明确要求不要复述原始版已知问题。
- **人工把关**：Codex 跑在沙箱里，不能 bind TCP、没装 pytest，所以它有几条只能「逻辑推导」。**我们在真机上把每一条可验证的都复现了一遍**，尤其是它推导但没跑通的 R09（符号链接逃逸）——我们实测吐出了根外文件，坐实了它。

## 三、两条最该记住的

### R09——重构自己引入的路径穿越，而且第一轮还夸过它

第一轮的「经核实不成立」里，我们信心满满地写下「符号链接逃逸防护有效」。那个结论对**直接**放在 `/static/` 下的符号链接是对的。但 `_handle_static` 处理目录索引时：

```python
if path.is_dir():
    path = path / "index.html"   # ← 追加后没有再 resolve/校验
```

`resolve_static_path` 只验证了**目录本身**在根内；追加的 `index.html` 若是符号链接，`read_bytes()` 会跟着它逃出去。实测 `GET /static/docs`（`docs/index.html -> ../../secret.txt`）返回 200 + 根外文件内容。

修复是追加 index 后再 `resolve()` + `relative_to(static_root)`，逃逸返回 403。

> **教训**：「防护有效」这种结论，有效范围只等于你实测覆盖过的输入形状。换一条路径（嵌套目录 index）它就塌了。第一轮四个审查者都没走到这条路径，因为原始代码的 `serve_static` 结构不同——**是重构改变了代码形状，也就改变了攻击面**。

### R05 + R08——「求稳」反而制造了 500

这两条是同一个模式，值得单独点出。重构时我们特意「变严」：
- 用 `isdigit()` 校验 Content-Length（比裸 `int()` 稳）；
- 用 `secrets.compare_digest()` 比对密码（防时序侧信道）。

结果两个「更稳」的原语都在**非 ASCII 输入**上翻车：`"²".isdigit()` 为真但 `int("²")` 抛异常；`compare_digest` 对非 ASCII `str` 直接抛 `TypeError`。两者都把**客户端错误变成了服务端 500**——正是我们在原始代码里批评过的「错误分类失真」，只是方向反了。

> **教训**：引入一个「更安全/更严格」的库函数时，要问它在**边界输入**上的行为，而不是只看它在正常输入上更好。稳的原语用错了地方一样会炸。

## 四、测试

新增 19 个回归测试，逐条锁定上述修复；总数 **61 → 80**。

- 单元测试：38 → 55（新增空/非法 Content-Length、非 ASCII 数字、冒号前空白、响应头大小写、Host 校验、rotate 过期会话、非 ASCII 凭据、嵌套 index 符号链接、头部上限只算头部、通用异常返回固定 500 不泄漏）。
- 功能测试：23 → 25（新增 semaphore 在 accept 前限流、取消时 socket 关闭）。

R16 的三条「名不副实」测试也一并订正：
- **R16.1**：原大小写测试只测精确大小写，漏掉小写 `content-length` 重复（正是 R07 的洞）→ 新增大小写无关断言。
- **R16.2**：`test_sleep_without_param` 实际发的是 `sec=0`，绕过了无参默认分支 → 改为真正不带参数，断言默认 1 秒。
- **R16.3**：泄漏测试只触发显式 400 校验，没走通用异常兜底 → 新增注入崩溃 handler 的测试，断言 500 固定文案且不泄漏异常原文。

全部 80 项在 macOS 上三连跑稳定通过。

## 五、这一轮对整个练习的意义

把两轮连起来看，最值钱的一课不是任何单条 bug，而是：

> **每一层「已验证」都有它的有效边界，跨过边界就得重新验证。**
>
> - 原始测试全绿 ≠ 代码对（第一轮的发现）。
> - 审查原始代码 + 重构 + 新测试全绿 ≠ 重构对（这一轮的发现）。
> - 「路径穿越防护有效」≠ 换个路径形状也有效（R09）。
> - 「换了更稳的库函数」≠ 边界输入也更稳（R05/R08）。

AI 审查能帮你逼近这些边界，但**边界之外要不要信、信到什么程度，得人工在真机上复现了才算数**。这一轮我们没有照单全收 Codex 的 16 条——R15 已修作废、R13/R10/R11 按性质分级处置，R09 则是我们比 Codex 更进一步、把它只能推导的东西真跑通了。
